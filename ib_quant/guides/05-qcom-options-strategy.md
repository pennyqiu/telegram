# QCOM 期权卖方策略 + 真实数据工具（整理版）

> 本文把"QCOM 系统化卖 Put 策略"从设计、真实数据获取、到工具落地的全过程整理成一份可复用文档。
> 配套脚本：
> - `tools/options_data.py` —— 公开数据源（CBOE）版，**无需 IB 账户，现在就能用**
> - `tools/qcom_options.py` —— IB Gateway 版，账户开通后取**真实保证金**（whatIf）
>
> ⚠️ 仅供技术研究，不构成投资建议。期权卖方具非线性、潜在大额亏损特征。

---

## 1. 背景与约束

| 项目 | 设定 |
|---|---|
| 账户净值 (NLV) | $200,000 |
| 账户类型 | Portfolio Margin (PM)，资本效率高，但压力测试下维持保证金敏感 |
| 标的 | 高通 QCOM |
| 核心诉求 | 规则化、可滚动、**尾部风险优先**的卖 Put / Wheel 策略 |

**硬约束**

1. 单资产最大名义敞口 ≤ 1.5× NLV（即 ≤ $300,000）。
2. 初始保证金 (IM) 利用率控制在 **20%–30%**，留足应对 IV Spike / 黑天鹅。
3. 偏好 **|Delta| 0.15–0.25** 的虚值 Put 建仓与滚动，重 Theta 现金流与安全边际。

---

## 2. 市场环境诊断（关键，决定打法）

基于真实数据（CBOE，2026-06-26 收盘）：

| 指标 | 真实值 | 含义 |
|---|---|---|
| QCOM 现价 | **$188.70**（当日 −7.57%，盘中振幅 ~15%） | 高实际波动 |
| ATM / 近月 IV | **~72–77%**，IV Rank ~70、Percentile ~88 | 期权"很贵"，历史高位 |
| HV20 | **~86–95%** | **HV > IV → 波动率风险溢价(VRP)为负** |

**结论性立场**

- **负 VRP（HV>IV）是当前最大红灯**：实际波动比卖期权拿到的对价还大，裸卖被低估补偿。
- 因此采用 **"远价外 + 尾部对冲 + 克制规模"**，而非满仓裸卖。
- 高 IV 的好处：0.20Δ 的 Put 自然落在 ~$160（约 15% 价外），**安全垫很厚**，且权利金丰厚。

## 3. 策略架构

### 3.1 两种结构模式

| 模式 | 结构 | 适用 |
|---|---|---|
| **naked**（裸卖+尾部对冲） | 卖 0.15–0.20Δ Put（Margin-Secured Put）+ 买 ~0.06Δ 深虚值 Put 封尾部 | 资本效率优先；IV 高位博崩塌回吐 |
| **spread**（牛市看跌价差） | 卖 0.15–0.20Δ Put − 买低 N 点行权价 Put（定义风险） | **尾部风险优先 / 负 VRP 环境（推荐当前用）** |

### 3.2 参数

| 参数 | 设定 | 机理 |
|---|---|---|
| 开仓 DTE | 35–50 天 | Theta 效率与 Gamma 风险最优平衡 |
| 短腿 \|Δ\| | 0.15–0.20 | 高 IV 下安全垫厚 |
| 对冲腿 \|Δ\| | 0.05–0.06 | 便宜尾部保险 / 释放 PM 压力保证金 |
| 价差宽度 | 15–20 点 | 定义最大风险 |
| 入场节奏 | 分 2–3 批 | 暴跌日避免单点接刀 |

### 3.3 头号风险：Vega 而非 Delta

高 IV（72%）基数上，+50% IV Spike = +35 个波动点，组合净 Vega ≈ −$150/点 → Vega 才是压力情景的主要亏损来源。尾部/保护腿的正 Vega 是关键缓冲。

| 净 Vega | −$157/点 | −$148/点 |

**洞察**：当前负 VRP + 高 IV 环境下，**方案②（价差）更契合"尾部风险优先"**——收益更高、黑天鹅浮亏更小、且最大风险是确定值。代价是放弃一点 Theta 效率。

## 5. 风控与滚动规则

### 5.1 保证金 / 压力应对

- 正常环境目标 IM 利用率 20–30%，闲置缓冲 ≥ 70%。
- 压力情景（−20% / +50%IV）：动用缓冲优先 → IM>45% 平最近到期短腿 → IM>55% 或回撤>12% 执行 Roll Out & Down → 累计实亏达 NLV 8%（$16,000）整体减仓 50%。

### 5.2 止盈 / 滚动

| 触发 | 动作 |
|---|---|
| 达 **50% 顶格利润** | 买回平仓落袋 |
| **DTE 剩余 21 天** | 滚动到下一个 35–50DTE 周期，规避末端 Gamma |
| 短腿 **Δ 触 0.40–0.45** 或股价触行权价 | **Roll Out & Down，严守净 credit**；不能收 credit 则减仓不硬扛 |

### 5.3 被行权后 → Wheel 轮转

被动接股后转 Covered Call（Δ 0.20–0.25、35–45DTE、行权价 ≥ 成本），叠加季度股息（$0.92）摊低成本，完成 Put → 股票 → Call → 现金 的循环。

## 6. 数据获取方案（为什么这样选）

| 来源 | 状态 | 说明 |
|---|---|---|
| Yahoo / yfinance | ❌ 弃用 | 对该 IP 限流严重（Too Many Requests） |
| **CBOE 延迟报价** | ✅ **当前使用** | 免费、无需登录，**自带 Delta/Gamma/Theta/Vega/IV**，实测现价与行情一致 |
| IB Gateway (whatIf) | ⏳ 待账户开通 | 唯一能取**你账户真实保证金**的来源 |

> 局限：CBOE 不提供"你账户的保证金"。naked 模式保证金为 BS 模型估算（偏乐观）；spread 模式最大风险=（宽度−净权利金）为**精确值**。真实保证金待账户下来用 `tools/qcom_options.py` 的 whatIf 复核。

## 7. 工具使用手册

### 7.1 公开数据源版（无需 IB）：`tools/options_data.py`

依赖：Python 3.10+（已用 `urllib`/`requests`，无重依赖）。

```powershell
cd f:\cursor\telegram\telegram\ib_quant
$env:PYTHONIOENCODING="utf-8"          # Windows 让中文正常显示

# 裸卖保守版 + 压力测试
python tools/options_data.py --stress --max-notional 1.0

# 牛市看跌价差（定义风险）版 + 压力测试
python tools/options_data.py --mode spread --spread-width 20 --stress

# 同时推送到 Telegram
python tools/options_data.py --stress --max-notional 1.0 --notify
```

**常用参数**

| 参数 | 默认 | 说明 |
|---|---|---|
| `--symbol` | QCOM | 标的代码 |
| `--nlv` | 200000 | 账户净值 |
| `--mode` | naked | `naked` / `spread` |
| `--spread-width` | 20 | 价差模式保护腿低于短腿的点数 |
| `--dtes` | 35 49 | 目标到期天数（自动找最接近的真实到期） |
| `--short-deltas` | 0.18 0.16 | 各到期卖出腿目标 \|Δ\| |
| `--hedge-delta` | 0.06 | 尾部对冲腿 \|Δ\|（naked 模式） |
| `--target-im` | 0.25 | 目标初始保证金利用率 |
| `--max-notional` | 1.5 | 名义敞口上限（× NLV） |
| `--stress` / `--stress-spot` / `--stress-iv` | off / −0.20 / 0.50 | 压力测试及情景幅度 |
| `--notify` | off | 推送到 Telegram |

### 7.2 开启 Telegram 推送

在 `ib_quant/.env` 设（复用项目同一个 Bot 即可）：

```
TELEGRAM_BOT_TOKEN=你的token
TELEGRAM_CHAT_ID=你的chat_id
```

`--notify` 逻辑：优先走项目 `notifier`（同时触发邮箱等已注册通道）；若该模块/依赖不可用，则自动回退到只用上面两个变量直连 Telegram Bot API（无需额外安装 python-telegram-bot）。

### 7.3 IB 真实保证金版（账户开通后）：`tools/qcom_options.py`

只读连接 IB Gateway，用 `whatIfOrder` 取**真实**初始/维持保证金（PM 抵消后真值）。

```powershell
python tools/qcom_options.py --md-type 3   # 3=延迟行情, 1=实时(需订阅)
```

前提：IB Gateway/TWS 已登录并开启 API（端口：模拟盘 4002 / 实盘 4001）。
