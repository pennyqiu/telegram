第一章 战略定位与基本盘

1.1 公众号在三层漏斗中的角色

小红书(顶部破圈)→ 知乎(中部深度)→ 公众号(底部信任孵化)→ 朋友圈(私域复利)→ 1V1 签约
维度 公众号在矩阵中的独特价值
承接位置 知乎万字研报的「沉淀地」;小红书钩子的「兑换券」最终归宿
读者画像 已经看过你 2–3 篇知乎/小红书内容,带着"想多了解你"的意图来的中后段用户
决策影响 客户加你微信前的最后一道信任验证;决定他是否愿意进入 KYC 流程
内容时长 唯一允许 5000 字+ 完整论证的私域阵地(朋友圈太短、小红书太短、知乎偏公域)

1.2 三大写作原则

  1. 每篇必须能独立成单:不依赖前文,新关注者随便点开一篇都能拿到价值
  2. 每篇必须有一个反直觉点:CPA 视角的"冷峻拆解"是核心壁垒,不是科普文
  3. 每篇都要为「未来某位翻看历史的客户」而写:3 年后翻看依然有信息价值

第四章 选题方法论(M7 之后自己选题用)

4.1 5 个选题钩子公式

公式 范例 适用场景
「X 不是 Y」反共识 大厂股票不是"安全资产" 认知升级类
「N 步法 / N 维度」结构化 4 种教育金工具横评 认知升级 + 产品横评
「我是如何帮 XX 的」案例 我是如何帮一位字节 M5… 匿名案例类
「90% 人误解 / 7 年混淆」 CRS 申报 ≠ 征税 政策解读类
「N 个真实场景 / 陷阱」 PR 身份反向卡住的 3 个场景 政策 + 案例混合

4.2 每篇必含的 5 个结构件

  1. 开篇钩子(200 字内):一个反直觉问题 + 一个具体数字/场景
  2. CPA 视角声明(300 字):为什么这个问题非 CPA 视角说不清
  3. 正文论证(2500–4500 字):数据 + 表格 + 公式 + 图(图至少 3 张)
  4. 主动暴露劣势/边界(500 字):本方案适合什么、不适合什么
  5. 合规 CTA(200 字):3 类客户对号入座型,不留微信号(详见 § 4.4)

4.3 严禁出现的内容(合规红线)

完全沿用手册 § 7.1 的合规底线,公众号专属补充如下:

严禁 替代
文中出现具体公司股价、客户姓名、公司名称 一律使用"某 TMT 上市公司财务负责人"等去身份化表述
文中出现"立即购买 / 仅剩 / 优惠 / 返佣" "如需进一步了解,欢迎私信交流"
承诺收益数字 一律标注「非保证 · 基于精算假设 · 仅供参考」
留微信二维码或微信号 只留公众号回复关键词领取物料,微信号通过 1V1 私信发送
案例使用单一客户原型 必须多个真实样本归纳 + 改名 + 改地点 + 改数字
文末提供产品计划书 PDF 只提供方法论类物料(自查表 / 流程图)
