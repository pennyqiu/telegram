# 保险资讯周报系统 - 架构说明

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      数据源层                                  │
├─────────────────────────────────────────────────────────────┤
│  • 香港保监局 (IA)      • 内地银保监会                         │
│  • 保险公司官网          • RSS 新闻源                          │
│  • 知乎保险话题          • 微信公众号                          │

             │
             ▼
┌─────────────────────────────────────────────────────────────┐
│                    爬虫采集层 (crawler.py)                     │
├─────────────────────────────────────────────────────────────┤
│  • BaseCrawler       - 爬虫基类（HTTP请求、重试机制）          │
│  • RSSCrawler        - RSS 订阅解析                           │
│  • HKIACrawler       - 香港保监局新闻爬虫                      │
│  • ZhihuCrawler      - 知乎热门内容（需登录）                  │
│  • 手动收集工具       - manual_curator.py                     │
└────────────┬────────────────────────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────────────────────────┐
│                   内容分析层 (analyzer.py)                     │
├─────────────────────────────────────────────────────────────┤
│  • 自动分类           - 基于关键词匹配                         │
│  • 关键词提取         - 识别核心主题                           │
│  • 相关度计算         - 根据用户兴趣过滤                       │
│  • 去重               - 基于标题相似度                         │
└────────────┬────────────────────────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────────────────────────┐
│                   简报生成层 (briefing_generator.py)           │
├─────────────────────────────────────────────────────────────┤
│  • HTML 生成          - 美观的响应式页面                       │
│  • 章节组织           - 按分类整理内容                         │
│  • 历史归档           - 自动保存快照                           │
│  • 索引页面           - 方便浏览历史简报                       │
└────────────┬────────────────────────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────────────────────────┐
│                      输出层                                    │
├─────────────────────────────────────────────────────────────┤
│  • weekly.html                  - 最新一期周报                │

└─────────────────────────────────────────────────────────────┘
```

## 数据流

```
输入 → 爬取 → 清洗 → 分析 → 分类 → 生成 → 输出

1. 配置数据源 (config.py)
   ↓
2. 多线程爬取文章 (crawler.py)
   ↓
3. 标准化数据格式 (Article 对象)
   ↓
4. 内容分析和分类 (analyzer.py)
   ↓
5. 生成 HTML 简报 (briefing_generator.py)
   ↓
6. 保存文件并创建索引
   ↓
7. (可选) 自动部署到服务器
```

## 核心对象

### Article (文章对象)
```python
Article {
    id: str              # 唯一标识
    title: str           # 标题
    url: str             # 原文链接
    source: str          # 来源
    published: datetime  # 发布时间
    summary: str         # 摘要
    category: str        # 分类
    keywords: List[str]  # 关键词
}
```

## 配置文件

### config.py
```
REGULATORY_SOURCES   - 监管机构数据源
NEWS_SOURCES         - 新闻媒体数据源
SOCIAL_SOURCES       - 社交媒体数据源
DATA_SOURCES         - 数据报告来源
KEYWORDS             - 分类关键词字典
CRAWLER_CONFIG       - 爬虫行为配置
BRIEFING_CONFIG      - 简报格式配置
```

## 使用流程

### 开发阶段
```bash
# 1. 测试演示
python3 demo.py

# 2. 手动添加优质内容
python3 manual_curator.py

# 3. 配置数据源
vim config.py

# 4. 生成真实简报
python3 briefing_generator.py
```

### 生产部署
```bash
# 1. 配置定时任务（每周一早上8点）
0 8 * * 1 cd /path/to/insurance_briefing && python3 briefing_generator.py --output /var/www/insurance-briefing

# 2. 配置 Nginx 托管
location /insurance-briefing/ {
    alias /var/www/insurance-briefing/;
    index weekly.html;
    auth_basic "保险简报";
    auth_basic_user_file /etc/nginx/.htpasswd;
}

# 3. 访问
https://your-domain.com/insurance-briefing/
```

1. 在 `config.py` 中配置
2. 在 `crawler.py` 中实现爬虫类
3. 在 `crawl_all_sources()` 中调用
4. 测试并提交
