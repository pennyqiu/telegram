11 篇 CPA 深度研报 · 知乎发布 SOP

适用范围：把 insurance-guide/articles/cpa-*.html 的 11 篇深度研报搬运到知乎个人主页。
配套素材：转换后的 Markdown 文件在 insurance-guide/zhihu-md/，转换脚本在 convert_to_zhihu_md.py。

一、推荐工作流（最稳，损耗最小）

Step 1 · 用 Markdown 编辑器中转

知乎编辑器对直接粘贴 Markdown 源码支持有限（部分格式会被识别为纯文本）。最稳的做法是经过 Markdown 编辑器渲染成富文本后再粘贴。

推荐编辑器（任选其一）：
- Typora（macOS/Windows，付费 $14.99，最丝滑）— 所见即所得
- Mark Text（免费、开源、跨平台）
- VS Code + Markdown Preview Enhanced 插件（免费）
- Obsidian（免费）
- 石墨文档 / 飞书文档 / Notion（在线，免安装，可直接渲染 Markdown）

Step 2 · 4 步搬运流程

1. 用 Markdown 编辑器打开 zhihu-md/cpa-01-dividend-trap.md
2. 切换到「预览模式」或「阅读模式」（看到渲染后的富文本）
3. Ctrl+A 全选 → Ctrl+C 复制
4. 知乎写文章 → 在编辑器里 Ctrl+V 粘贴

预期效果：
- ✅ 标题层级（H1–H4）正确保留
- ✅ 加粗、斜体、列表（含嵌套）保留
- ✅ 表格保留（知乎自动渲染）
- ✅ 引用块保留
- ✅ 代码块保留（知乎渲染为深色背景）
- ✅ 链接保留

Step 3 · 文章标题与封面

  • 知乎要求文章标题独立填写（不是从正文 H1 抓）。直接用 Markdown 文件的 H1（第一行的 # ...）作为标题。
  • 封面图建议自制：纯色背景 + 文章标题 + "CPA 深度研报 #N"标签。可用 Canva / Figma 一次性做 11 张 1200×675 的封面。

二、知乎编辑器的格式损耗清单

原 HTML 元素 Markdown 中的形态 粘贴到知乎后的效果 是否需要手动补救
H2 / H3 / H4 标题 ## / ### / #### ✅ 完美保留（知乎只到三级标题，H4 会降级为加粗段落） 无
加粗、斜体 **...** / *...* ✅ 完美保留 无
无序 / 有序列表（含嵌套） - / 1. ✅ 保留 无
表格 GFM 表格语法 ✅ 保留（行高较大、列宽自适应） 无
引用块 > ... ✅ 保留（左边竖线） 无
代码块 ``` ```text ✅ 保留（深色背景，支持横向滚动） 无
链接 [文字](url) ✅ 保留 无
分隔线 --- ✅ 保留 无
HTML 自定义彩色框（warn / danger / green / callout） > ⚠️ **劣势暴露：** ... ⚠️ 变为普通引用块（颜色丢失，但 emoji + 加粗标签让语义保留） 可选：高优先级提示可加截图配图
HTML 渐变背景 / 卡片阴影 无对应 ❌ 丢失 不影响阅读，不需补救
<div class="formula"> 深色公式框 ```text 代码块 ✅ 渲染为深色背景，效果几乎一致 无
文末 <p class="cite"> 数据源 *...* 斜体 ✅ 保留 无

知乎首段技巧（不要直接粘贴文件中的引用摘要）

知乎首段会被作为「文章列表预览」抓取。建议每篇文章手动改写首段为：
- 疑问句开头（触发好奇心，提高完读率）
- 不超过 80 字（手机端预览最多显示 2 行）
- 包含核心关键词（利于知乎搜索 SEO）

举例（第 1 篇）：

原文摘要（机械）：

这篇研报不试图说服你买或不买，而是把分红储蓄险的精算假设、费用结构、非保证收益的兑现机制全部摊开...

改写后（知乎首段）：

香港分红险到底是不是智商税？这个问题被问了 6 年，至今没有任何代理给过完整答案。本文用 CPA 视角 + 8500 字，把保单的精算假设、费用结构、非保证收益的兑现机制全部摊开，看完你自己就能判断。

四、合规与法律检查清单（发布前必看）

知乎平台规则（每次发布前自检）

  • [ ] 不含直接销售引导（如「私信咨询报价」「VX 加我」）
  • [ ] 不含具体公司收益保证语言（如「保 7%」「3 倍回报」）
  • [ ] 引用第三方数据 → 标明来源（hkinsu.com / 香港保监局 / Moody's 等）
  • [ ] 横评不诋毁同业（保持「只摆数据，不下价值判断」）
  • [ ] 案例均已脱敏（无真实姓名、公司、金额）
  • [ ] 文末没有微信号 / 手机号（合规底线）

HKIA 合规要求（同步遵守）

  • [ ] 不在内地公开发起销售或招揽行为
  • [ ] 仅作教育 / 科普 / 学术目的
  • [ ] 不承诺非保证收益
  • [ ] 不展示返佣信息（保监局明令禁止）

七、常见问题排查

Q1：知乎粘贴后表格变成横向单行文字？

A：知乎编辑器对 Markdown 表格的识别要求：表头分隔行 | --- | --- | 必须存在。脚本已自动生成，如果手动调整后丢失，需要补回。

Q2：粘贴后代码块没有深色背景？

A：知乎需要明确的代码语言标识（```text、```python 等）。脚本已统一使用 ```text，应该不会出问题。如果仍有问题，手动在知乎编辑器里选中文字，点工具栏的「代码块」按钮。

Q3：引用块的 emoji 显示成方块？

A：知乎 PC 端 Web 编辑器 emoji 渲染正常，但部分企业内网或老旧浏览器可能显示成方块。如果担心，可以把 emoji 替换为纯文字标签：
- ⚠️ **劣势暴露：** → 【劣势暴露】
- 🚨 **严重警告：** → 【严重警告】
- ✅ **合规建议：** → 【合规建议】
- 💎 → 【洞察】

Q4：内部链接 (cpa-XX.html) 怎么办？

A：脚本已自动把内部链接转成「xxx（见姐妹篇）」的形式。等你在知乎发布了所有 11 篇之后，可以手动把它们替换为知乎站内链接（zhihu.com/p/{文章 ID}）。
