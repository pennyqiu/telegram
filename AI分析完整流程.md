### 步骤3：AI批量分析（15-30分钟）

```bash

```


```bash
python3 ai_pdf_analyzer.py
```

**操作：**
1. 选择AI服务（推荐Ollama本地免费）

### 步骤4：人工复核（1-2小时）

**为什么需要复核？**
- AI可能误读PDF中的数字
- 某些条款需要专业理解
- 确保关键数据准确

**复核重点：**
```bash
# 打开AI分析的结果
cd ai_analyzed
open 盛利II储蓄保险_AI分析_20260520.json

# 对照原PDF检查
open ../insurance_pdfs/AXA/盛利II储蓄保险_20260520.pdf
```

**关键字段必须人工确认：**
- ✅ 年缴保费范围
- ✅ 保证收益率 vs 预期收益率
- ✅ 回本年期
- ✅ 退保损失率
- ✅ 免责条款

**快速复核技巧：**
1. 先看AI标记为"不确定"的字段
2. 重点核对数字类数据
3. 优缺点可以适当补充
4. 保存修改后的版本

### 步骤5：批量对比分析（10分钟）

```bash
# 将复核后的文件复制到工作目录
cp ai_analyzed/*.json ./

# 运行对比工具
python3 simple_compare.py
```

**对比功能：**
1. 产品摘要 - 快速浏览所有产品
2. 储蓄类对比 - 横向对比收益
3. 重疾险对比 - 对比保障
4. 导出Excel报告

**对比维度：**
- 费用：年缴、总保费
- 收益：预期收益率、回本期
- 风险：公司评级、分红实现率

## 🔧 故障排查

### 问题1：爬虫收集不到产品

**原因：** AXA网站结构可能变化

**解决：**
1. 选择"手动模式"，获取热门产品链接
2. 或使用浏览器开发者工具，手动复制链接
3. 填写到 `axa_download_list.json`

### 问题2：PDF下载失败

**原因：** 链接失效或需要认证

**解决：**
1. 在浏览器中测试链接
2. 如需登录，先登录后复制PDF链接
3. 某些PDF可能需要联系公司获取

### 问题3：AI分析失败

**原因：** API配置错误或模型未启动

**解决：**
```bash
# Ollama检查
ollama list  # 查看已安装模型
ollama serve # 启动服务

# OpenAI检查
echo $OPENAI_API_KEY  # 确认环境变量

# 如果都不行，使用模拟模式测试流程
```

### 问题4：PDF提取文本乱码

**原因：** PDF是图片扫描版

**解决：**
1. 使用OCR工具先转换
2. 或手动阅读PDF填写模板
3. AXA官方PDF通常是文本版，不应该有此问题
